# Skill: distill

## IRON LAW

**script.md 定节拍（口播顺序 = 视觉推进顺序），article.md 定画面密度（口播跳过的细节上屏幕）。永不混淆两个源的职责。**

outline.md 是跨步骤记忆载体，必须同时参考 script.md 和 article.md。

## Purpose

将原始内容（文章 / 转写文本）提炼为两个产出：口播稿（定节奏）和开发大纲（定画面）。这是整个流水线的核心转换步骤。

## Preconditions

- `plan.json` 已存在（由 using-recastory 生成）
- 输入文件存在：`raw/article.md`（或 `transcribe/transcript.json`）
- 如指定 `--perspective`，对应视角 SKILL.md 可用

## Steps

> distill 内部分为三个阶段，每个阶段有独立质量门控。阶段间不可合并执行。

---

### Phase A: 结构化阅读（不生成任何内容）

此阶段的目标是建立完整的心智模型。**不产出任何文件**，只在 Agent 上下文中形成理解。

#### A.1 读取计划

从 `plan.json` 获取：
- `double_source.article` — 原文路径
- `double_source.script` — 口播稿输出路径
- `double_source.outline` — 大纲输出路径
- `perspective.name` — 视角名称（如有）
- `perspective.expression_dna` — Expression DNA 摘要（如有）
- `target_format` — 目标格式（决定时长/字数）
- `register` — 注册表类型（brand/product）

#### A.2 读取原文

读取 `raw/article.md`。

提取并内部结构化：
- 核心论点（1-3 个，每个一句话）
- 关键数据 / 引用 / 案例（带原文出处）
- 可视觉化的元素（图表、对比、流程）
- 受众痛点 / 入口钩子（哪些点最能抓注意力）
- 原文叙事弧线（开头张力、中间证据、结尾收束）

#### A.3 视角深度注入（如 plan.json 指定 perspective）

根据 `perspective.source` 字段选择加载方式：

- **`source: "recastory"`** 或无 source 字段 → 调用 `Skill` 工具加载 `skills/perspectives/<name>/SKILL.md`
- **`source: "nuwa-skill"`** → 跳过 Skill 调用，直接使用已加载的上下文（问答式格式，提取表达风格特征）

从 SKILL.md 提取两个维度：

**Expression DNA**（现有维度）：
- 句式规则、词汇偏好、节奏模式、幽默策略

**Narrative Heuristics**（新增维度）：
- **章节排序策略**：先结论还是先例子？先震撼还是先铺垫？
- **重点分配策略**：多少比例给"为什么反直觉"vs"发生了什么"？
- **Hook 策略**：这个视角怎么开场？（数字震撼？反差提问？具体场景？）
- **结尾模式**：开放问题？呼应开头？悬念？（禁用总结式结尾）

如视角 SKILL.md 未显式定义 Narrative Heuristics，从 Mental Models 和 Expression DNA 推导。

如未指定 perspective，跳过此步，使用中性口语风格。

#### A.4 理解验证门（Comprehension Gate）

**Agent 必须在内部形成以下结构化理解**（不写入文件，只在上下文中明确）：

```
COMPREHENSION CHECK:
- 核心论点: [1 句话]
- 关键证据: [3-5 条]
- 可视觉化时刻: [列表]
- 视角 Hook 策略: [这个视角会怎么开场？]
- 叙事策略: [这个视角会怎么排章节？重点放在哪？]
- Register 对齐: [brand=戏剧化？product=信息密集？]
```

**门控规则**：如果 Agent 无法形成连贯的理解块，必须重新阅读原文。此门控确保真正的理解而非表面浏览。

---

### Phase B: 逐章生成（视角 + humanizer 主动约束）

此阶段生成 script.md。**生成约束在写之前加载，不是写完再检查。**

#### B.1 加载生成约束

在写任何文字之前，加载以下约束到上下文：

1. `references/content-distillation/REFERENCE.md` 的格式规则（句子长度、人称、节奏标记）
2. `references/content-distillation/REFERENCE.md` 的**生成约束 Group 1**（10 条：H-01/04/07/09/10/22/24 + SL-001/004/006）
3. 视角 Expression DNA（句式、词汇、节奏、幽默）
4. 视角 Narrative Heuristics（章节排序、重点分配、Hook、结尾）

合并为一份"生成约束清单"，在逐章生成时作为硬性规则遵守。

#### B.2 规划章节结构

根据 Phase A 的理解 + 视角 Narrative Heuristics：

- 确定章节数（从 target_format 推算：30s ≈ 1-2 章，60s ≈ 2-3 章，每增 30s +1 章）
- 将核心论点 / 证据簇分配到各章
- 按视角叙事启发式规划章节顺序和 Hook 策略
- **视角结构影响**：视角决定章节排序和重点分配（如 Feynman: 具体例子→反直觉→原理→开放问题；MrBeast: 极端数字→赌注→行动→升级→悬念）

#### B.3 逐章生成 + 即时检查

**逐章生成**（不一次性生成全文）：

对每一章：
1. 按生成约束清单 + 视角 DNA 写该章全文
2. 应用视角 Expression DNA（句式、词汇、节奏）
3. 应用视角 Narrative Heuristics（Hook 策略、重点比例）
4. 字数目标：30s = 150-200 字，60s = 300-400 字，每增 30s +150-200 字

**即时检查**（每章生成后立即执行）：

| 检查项 | 检测 | 严重度 |
|--------|------|--------|
| SL-001 | 假共情（"我知道你""你是不是也"） | critical |
| SL-003 | 自我标榜（"颠覆认知""你一定要听完"） | critical |
| SL-004 | 万能模板（"说白了/本质上/底层逻辑"） | critical |
| SL-006 | 套话结尾（"以上就是本期内容"） | critical |
| CD-003 | 书面语残留（"综上所述/由此可见"） | critical |
| DS-002 | 长句（单句 >20 字） | warning |
| 生成约束 | AI 词汇、填充短语、三段式、否定式排比等 | warning |

**门控**：0 critical violations 才进入下一章。
**修复方式**：定位违规句子，修复具体句子（不重写整章）。

#### B.4 组装 script.md

合并所有章节为完整文件。

**格式要求**：
- 按章节组织：`## 第N章：<标题>`
- 每章含 2-3 个步骤，用 `### 步骤 <N>` 标记

**口语化规则**（硬性，贯穿全文）：
- 每句 ≤20 字（B 站基准；抖音 ≤12、YouTube ≤30、知乎 ≤25）
- 第二人称为主（"你看"、"你想想"）
- 去填充词（"那么"、"其实"、"所以说"、"接下来"）
- 去书面语（"综上所述"→"所以你看"、"由此可见"→"你看吧"）
- 短句优先，允许不完整句（"真的。""你猜怎么着？"）
- 变化句式：长-短-长交替，避免连续 5 句相同结构

**组装后检查**：字数在目标 ±30% 内。

#### B.5 [Checkpoint: SCRIPT_PREVIEW]

script.md 组装后，向用户展示：
- 口播稿前 200 字（开场 Hook + 第一章）
- 字数统计（总字数 / 章节数 / 每章字数）
- SL-001~SL-006 自检结果（有无违规）
- 视角注入状态（已注入 / 跳过 / 未指定）
- **生成约束合规摘要**（Group 1 共 10 条约束的检查结果）

**必须收到用户确认才可继续。** 用户可要求：修改风格 / 调整字数 / 换视角 / 重新生成 / 继续。

---

### Phase C: 大纲 + 质量门控

此阶段生成 outline.md 并执行全量质量检查。

#### C.1 生成 outline.md

从 script.md 和 article.md **双源参考**，生成开发大纲。

**结构**：
```markdown
# Outline: <pipeline_id>

## 章节 1: <标题>（<N> 步骤，约 <M> 秒）
### 步骤 1
- **屏幕内容**：<hero 图 / 数据卡片 / 标语 / 列表项>
- **口播要点**：<script.md 中对应步骤的核心信息>
- **信息池**：<从 article.md 提取的补充细节，可挂到屏幕上>

### 步骤 2
...

## 章节 2: <标题>（<N> 步骤，约 <M> 秒）
...
```

**双源执行规则**：
- **从 script.md 取**：章节切分、每章步骤数、口播要点（定节奏）
- **从 article.md 取**：屏幕内容细节、信息池（数据/引用/案例）、视觉化元素（定密度）

**outline.md 边界**（必须写 vs 不可写）：

| 必须写 | 不可写 |
|--------|--------|
| 章节切分 + 每章步骤数 + 估时 | 具体动画类型 |
| 每步屏幕内容描述（hero/数据/标语/列表） | CSS 实现手段 |
| 章节级信息池（数字/引用/案例） | 时长数值（秒） |
| 口播要点摘要 | 微动细节 |

#### C.2 全量反模式检查

对照以下规则逐项检查产出：

**内容提炼反模式**（检查 outline.md）：

| 规则 | 检测 | 修复 |
|------|------|------|
| CD-001 | 大纲层级 >4 层 | 扁平化或拆分视频 |
| CD-002 | 单章节 >500 字（短视频） | 拆分子章节 |
| CD-004 | 关键词覆盖率 <30% | 补充核心术语 |
| CD-005 | 前 10 秒无 Hook | 添加开场钩子 |
| CD-006 | 指定了视角但未注入 Expression DNA | 重新注入 |

**AI Slop 反模式**（检查 script.md）：

| 规则 | 检测 | 修复 |
|------|------|------|
| SL-001 | 假共情 | 直接抛事实/钩子 |
| SL-002 | 假深刻 | 去掉转折直接说结论 |
| SL-003 | 自我标榜 | 直接说 |
| SL-004 | 万能模板 | 删掉或直接说内容 |
| SL-005 | 排比堆砌 | 保留 1~2 个，其余砍掉 |
| SL-006 | 套话结尾 | 用视角启发式设计结尾 |

**口播风格反模式**（检查 script.md）：

| 规则 | 检测 | 修复 |
|------|------|------|
| DS-001 | 信息保留度 <60% | 补充关键事实/案例 |
| DS-002 | 长句未拆（单句 >20 字） | 拆分短句 |
| DS-003 | 第三人称疏离 | 改为"你""我们" |
| DS-004 | 无钩子开头 | 添加 Hook |
| DS-005 | 结构词堆砌 | 融进过渡或砍掉 |
| DS-006 | 数字未翻译 | 转为感受或视觉对比 |

**自检流程**：先检查 → 发现问题 → 修复 → 再检查 → 通过后才向用户报告。**不可跳过修复直接报告。**

#### C.3 五维质量评分

基于 humanizer-zh 评分体系，对 script.md 做最终质量评估：

| 维度 | 评分标准 | 分值 |
|------|---------|------|
| 直接性 | 事实是否直接陈述？有无铺垫/软化？ | /10 |
| 节奏 | 句子长度是否变化？长短交替？ | /10 |
| 信任度 | 是否尊重读者智慧？有无过度解释？ | /10 |
| 真实性 | 听起来像真人说话吗？有观点/声音？ | /10 |
| 精炼度 | 还有可删减的内容吗？ | /10 |

**门控**：总分 >= 40/50 且单维度 >= 4/10。
**不通过** → 定位最弱维度 → 重写得分最低的章节 → 重新评分。

#### C.4 保留原文

`raw/article.md` 保留不删。不修改、不重命名。

#### C.5 建议运行审计工具

输出后建议用户运行确定性审计：

```bash
python -m tools.audit <workspace>/distill/
```

审计工具对 CD/SL 规则做确定性检测，补充 Agent 自检的盲区。

## Output

- `workspace/<id>/distill/script.md` — 口播稿
- `workspace/<id>/distill/outline.md` — 开发大纲
- `raw/article.md` — 原文（保留不删）

## Anti-Patterns

| ID | 名称 | 检测 | 严重度 |
|----|------|------|--------|
| CD-001 | 层级过深 | outline 层级 >4 | warning |
| CD-002 | 章节过长 | 单章 >500 字 | warning |
| CD-003 | 书面语残留 | script.md 含 "综上所述/由此可见" 等 | critical |
| CD-004 | 信息密度低 | 关键词覆盖率 <30% | warning |
| CD-005 | 缺乏 Hook | 前 10 秒无吸引力 | warning |
| CD-006 | 视角未注入 | 指定了 perspective 但 script 风格中性 | critical |

### 确定性检测

反模式 CD/SL 规则可通过 `python -m tools.audit <workspace-dir>` 运行确定性检测。Agent 自检后，建议用户运行审计工具做二次验证。

## Failure Modes

| 场景 | 回退 |
|------|------|
| article.md 为空或不存在 | 阻断，要求用户提供输入 |
| 视角 SKILL.md 不存在 | 跳过视角注入，在 plan.json 标记，继续执行 |
| script.md 字数超出目标 ±30% | 警告用户，建议调整 |
| 反模式 CD-003/CD-006 触发 | 阻断，必须修复后才可继续 |
