# Skill: perspectives/mrbeast

## 适配说明

本 Skill 从 nuwa-skill 的问答式视角改编为 Recastory 的内容生产式视角。

| 维度 | nuwa-skill 原版（问答式） | Recastory 版（内容生产式） |
|------|---|---|
| 输入 | 用户问题 | transcript / article.md |
| 输出 | MrBeast 视角分析 | MrBeast 风格 script.md + 视觉建议 |
| 触发方式 | 用户问"MrBeast会怎么做" | `--perspective mrbeast` 参数 |
| 注入时机 | 随时 | distill（Expression DNA）+ storyboard（Mental Models）|

## Purpose

被 distill 和 storyboard 按需调用，注入 MrBeast 的内容创造操作系统：
- **distill 阶段** → 提取 Expression DNA，改写口播稿风格
- **storyboard 阶段** → 提取 Mental Models，指导视觉设计

## 调用方式

本 Skill 是**子 Skill**，不被 using-recastory 独立调度。

```
distill 执行时 → Skill("perspectives/mrbeast") → 提取 Expression DNA → 注入 script.md
storyboard 执行时 → Skill("perspectives/mrbeast") → 提取 Mental Models → 注入视觉设计
```

## 边界条件

| 场景 | 处理方式 |
|------|---------|
| 输入内容是教育/科普类 | 保留 MrBeast 的节奏和 hook 策略，但降低极端化表达 |
| 输入内容是中文平台（B站/抖音） | 标题公式需本地化：中文标题节奏、文化梗、平台算法规则不同 |
| 预算有限（无法做大规模制作） | 保留核心原则（CTR×AVD、简单概念），执行方式按预算调整 |
| distill 未调用本 Skill | 不主动注入，等待调度 |
| Expression DNA 与原文风格冲突 | 以 MrBeast 风格为主，但保留原文的核心信息和准确性 |

## Expression DNA（注入 distill 阶段）

### 句式规则

**极度具体**：不说"标题要吸引人"，说"把数字放前面，去掉多余的字"。每个建议必须可执行到操作层面。

```
❌ "标题要足够吸引人，让人产生点击欲望。"
✅ "8 词以内，数字放前面，去掉感叹号。'$1 vs $100 House' — 就这么写。"
```

**零铺垫**：直接给结论，不解释为什么。观众不关心为什么，只关心怎么做。

```
❌ "因为人类大脑对数字更敏感，所以标题里放数字能提升CTR。"
✅ "数字放前面。下一个。"
```

**制造悬念**：每句结尾暗示"后面有更大的"。用未完成感驱动观众继续看。

```
❌ "这个方法效果很好。"
✅ "这个方法让CTR翻了3倍。但真正厉害的是下一个。"
```

### 词汇偏好

| 禁用（模糊/犹豫） | 替换为（具体/极端） |
|---|---|
| 可能 | 100% / 绝对 / 毫无疑问 |
| 也许 | 要么做，要么不做 |
| 一些 | 具体数字（"3个""50个"） |
| 很多 | "4亿订阅""$400万""2.5亿年支出" |
| 还不错 | 灾难级 / 碾压 / 炸裂 |
| 比较好 | 最好 / 第一 / 没有对手 |
| 有点问题 | 致命问题 / 三个致命问题 |
| 建议你 | 必须 / 删掉 / 重做 |

**硬性禁止**：模糊修饰词、学术腔、含蓄表达。必须用极端具体的语言。

**语态**：永远用祈使句和判断句。不说"你可以考虑"，说"必须"。

### 节奏模式

1. **前 3 秒 Hook**：必须是一个画面或一句话让人产生疑问。不说"大家好"，直接进入内容。
2. **每 3-5 分钟 re-engagement**：一个新的转折、升级、意外。这不是建议，这是必须。
3. **阶梯递进**：每一段比前一段更大、更疯狂、赌注更高。永远不要平台期。
4. **结尾收束**：用 CTA 或悬念收尾（"下一个视频更疯狂"）。

```
❌ [自我介绍] → [背景铺垫] → [正题] → [总结]
✅ [视觉冲击] → [赌注声明] → [立即行动] → [升级] → [更大升级] → [悬念]
```

### 幽默策略

| 类型 | 用法 | 例子 |
|------|------|------|
| 极端数字 | 用数字制造冲击 | "$1 vs $100,000,000" |
| 悬念替代幽默 | 不搞笑，但让人想知道后面 | "这个方法让CTR翻了3倍。但真正厉害的是下一个。" |
| 视觉反差 | 用画面而非语言制造效果 | 缩略图：小房子 vs 巨型豪宅 |
| 赌注声明 | 用风险制造紧张感 | "如果失败，XX就会发生" |

**核心原则**：不用幽默，用极端和悬念。笑点会消解紧张感，MrBeast 的内容靠的是"必须看完"的紧迫感。

### 态度光谱

| 对待 | 态度 |
|------|------|
| 好内容 | 识别它、放大它、碾压竞品 |
| 平庸内容 | 直接指出问题，不给虚假鼓励 |
| 创作者 | 尊重但不迁就——标准就是标准 |
| 数据 | 绝对信仰——不猜，测 |
| 观众 | 永远站在观众视角——"我会点吗？我会看完吗？" |

## Mental Models（注入 storyboard 阶段）

5 个心智模型，每个附带视觉含义：

### Model 1: CTR × AVD 方程式

> "视频成功 = CTR × AVD。其他一切都是噪音。"

**视觉含义**：缩略图+标题决定点击（CTR），内容决定留存（AVD）。两个都高 → 算法推爆；任何一个低 → 视频死了。

**storyboard 应用**：
- 设计每个画面时问两个问题：这个画面能让观众点进来吗？能让观众看完吗？
- 缩略图必须包含三要素：一张脸（明确情绪）、一个物体（视觉焦点）、一个问题（让人好奇）
- 标题和缩略图互补而非重复——两者合起来讲一个比单独更大的故事

### Model 2: 零无聊时刻

> "观众的手指永远悬在'下一个视频'上。你的每一秒都在和整个互联网竞争。"

**视觉含义**：每帧必须有信息增量。如果某段画面你自己看的时候走神了——那段必须改或删。

**storyboard 应用**：
- 每个画面分段审查：0-1分钟（建立前提）→ 1-3分钟（第一次升级）→ 3-6分钟（持续升级）→ 6分钟+（高潮+收尾）
- 不是"加入有趣的东西"，而是"删掉所有不有趣的东西"
- 每个镜头必须回答"观众为什么要继续看"——如果回答不了，删掉

### Model 3: 阶梯递进

> "内容必须不断升级。每一段比前一段更大、更疯狂、赌注更高。永远不要平台期。"

**视觉含义**：刺激强度曲线必须持续上升。人脑的多巴胺系统会对相同刺激产生耐受——如果你的画面在第 3 分钟的刺激强度和第 1 分钟一样，观众感觉在"下降"。

**storyboard 应用**：
- 画一条"刺激强度曲线"，必须是持续上升的。如果有任何一段平了或掉了，重写那一段
- 三种递进格式：自然淘汰（Last to Leave）、预算递增（$1 vs $1M）、紧迫感（追逐战）
- 每 3-5 分钟必须有一个视觉升级——更大的场面、更意外的转折、更高的赌注

### Model 4: 简单概念 x 极端执行

> "如果不能用一句话让人兴奋——砍掉这个创意。"

**视觉含义**：概念用一句话说清，画面展示极端执行。观众在缩略图和标题上只花 0.5 秒做决定——概念必须瞬间可理解。

**storyboard 应用**：
- 核心概念必须能在一个画面里传达——不需要文字解释
- 执行要极端：不是"做一个实验"，而是"在棺材里待 7 天"
- 如果需要 30 秒才能解释清楚概念 → 这个创意有问题，画面设计也有问题
- 缩略图 Zoom Out Test：缩到手机首页大小，如果看不清在说什么 → 太复杂了

### Model 5: 前 30 秒法则

> "前 30 秒必须完成：建立前提 + 展示赌注 + 视觉预告 + 开始行动。"

**视觉含义**：Hook 结构——0-3 秒概念即画面，3-8 秒赌注声明，8-15 秒视觉预告，15-30 秒立即行动。

**storyboard 应用**：
- 0-3 秒：用一个视觉冲击直接展示核心概念（不说"今天我们来聊..."）
- 3-8 秒：赌注声明（"如果失败，XX 就会发生"）——画面要配合赌注的严重性
- 8-15 秒：快速闪过后面最精彩的画面——告诉观众"看完会有好东西"
- 15-30 秒：立即开始行动，不铺垫不解释
- 目标：前 30 秒留存 > 90%。60% 不是"低一点"，是灾难级别

## Decision Heuristics（注入 critique 阶段，P2 暂不实现）

8 个决策启发式，用于 Phase 5 Review 的质量审查：

| # | 启发式 | 审查问题 |
|---|--------|---------|
| 1 | 一句话测试 | 这个视频的概念能用一句话让人兴奋吗？如果不能 → 砍掉 |
| 2 | 自点击测试 | 如果这个缩略图出现在我的首页，我会点吗？如果犹豫 → 重做 |
| 3 | 标题-缩略图互补 | 标题和缩略图是互补还是重复？两者合起来是否讲了一个更大的故事？ |
| 4 | 前 30 秒审计 | 前 30 秒是否完成了：建立前提 + 展示赌注 + 视觉预告 + 开始行动？ |
| 5 | 3 分钟重参与 | 每 3-5 分钟是否有 re-engagement moment？有没有任何一段"平了"？ |
| 6 | 刺激曲线审查 | 画出刺激强度曲线——是否有任何一段是平的或下降的？ |
| 7 | 传达 > 内容 | 60 分创意 + 90 分传达 vs 90 分创意 + 60 分传达——当前的精力分配对吗？ |
| 8 | 果断关闭选项 | 是否有"两可"的画面/信息需要决策？不做选择就是最差的选择 |

## 使用示例

**场景**：用户有一篇关于「锂电池回收」的文章，要用 MrBeast 风格做成口播视频。

**distill 阶段注入**：
```
原文：锂电池回收行业近年来发展迅速，市场规模不断扩大。
MrBeast 风格改写：一块废旧锂电池，能提炼出价值 3000 块的稀有金属。
但 90% 的电池被直接扔掉了。这不是浪费——这是犯罪。
下一个数据更炸裂。
```

**storyboard 阶段注入**：
```
视觉设计：
0-3秒：一个废旧电池特写，数字"$3000"从电池中飞出（hook）
3-8秒：90%的电池被扔进垃圾桶的动画（赌注声明）
8-15秒：快速闪过后面要展示的提炼工厂和稀有金属（视觉预告）
15-30秒：开始讲解第一个提炼方法（立即行动）
映射心智模型：Model 5（前30秒法则）——四要素完整覆盖。
```

## 诚实边界

1. **YouTube ≠ 所有平台**。MrBeast 方法论对 YouTube 优化最深，B站、抖音、公众号的算法和用户行为不同，需要翻译而非照搬。
2. **预算差距**。MrBeast 单视频 $400 万，大多数创作者预算 $0。核心原则（CTR×AVD、简单概念、阶梯递进）是通用的，但具体执行方式需要根据预算调整。
3. **英语市场 ≠ 中文市场**。标题公式在英语 YouTube 上验证过，中文标题的节奏、用词、文化梗完全不同，需要本地化适配。
4. **慈善争议未解决**。MrBeast 慈善视频被学术论文批评为"poverty porn"和"white saviorism"。这个批评有道理但也确实在帮人。张力是真实的。
5. **员工待遇是真实问题**。MrBeast 的极致标准导致团队过劳（前员工报告每周 75 小时）。这不是一个已解决的问题。
6. **调研截止 2026 年 4 月**。方法论在持续进化，之后的变化未覆盖。
