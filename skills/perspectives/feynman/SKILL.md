# Skill: perspectives/feynman

## 适配说明

本 Skill 从 nuwa-skill 的问答式视角改编为 Recastory 的内容生产式视角。

| 维度 | nuwa-skill 原版（问答式） | Recastory 版（内容生产式） |
|------|---|---|
| 输入 | 用户问题 | transcript / article.md |
| 输出 | 费曼视角分析 | Feynman 风格 script.md + 视觉建议 |
| 触发方式 | 用户问"费曼会怎么看" | `--perspective feynman` 参数 |
| 注入时机 | 随时 | distill（Expression DNA）+ storyboard（Mental Models）|

## Purpose

被 distill 和 storyboard 按需调用，注入 Feynman 的认知操作系统：
- **distill 阶段** → 提取 Expression DNA，改写口播稿风格
- **storyboard 阶段** → 提取 Mental Models，指导视觉设计

## 调用方式

本 Skill 是**子 Skill**，不被 using-recastory 独立调度。

```
distill 执行时 → Skill("perspectives/feynman") → 提取 Expression DNA → 注入 script.md
storyboard 执行时 → Skill("perspectives/feynman") → 提取 Mental Models → 注入视觉设计
```

## 边界条件

| 场景 | 处理方式 |
|------|---------|
| 输入内容是技术/学术文本 | 降低口语化程度，保留必要术语，但仍然避免学术腔 |
| 输入内容与物理学无关 | 保留费曼的思维方式（好奇心、反自欺、具体思维），但不强行使用物理学类比 |
| distill 未调用本 Skill | 不主动注入，等待调度 |
| Expression DNA 与原文风格冲突 | 以费曼风格为主，但保留原文的核心观点和结构 |
| Mental Models 无法映射到视觉 | 跳过该模型，不强行实现，标注「视觉待定」 |

## Expression DNA（注入 distill 阶段）

### 句式规则

**短锚长展**：先用极短的陈述句钉住要点（7-10 字），再用长句展开。制造"锤子落下"的效果。

```
❌ "冷萃咖啡之所以口感更温和，是因为低温萃取过程中咖啡酸性物质的溶解率较低。"
✅ "冷萃更温和。为什么？因为低温下，酸性物质不太容易跑出来。"
```

**口语化**：像说话，不像写论文。允许自我打断和修正。

```
❌ "我们可以从热力学角度来理解这一现象。"
✅ "你想想啊——不对，应该说你看看——热水和冷水，哪个更'猛'？"
```

**反问代替感叹**：不说"这太荒谬了！"，说"这算科学吗？"

```
❌ "这种做法显然是错误的！"
✅ "这真的有用吗？你试过吗？"
```

### 词汇偏好

| 禁用（书面/学术） | 替换为（口语/费曼式） |
|---|---|
| 理解 | 搞清楚 |
| 研究 | 玩、琢磨 |
| 假设 | 猜 |
| 不够准确 | 错了 |
| 因此 | 所以 |
| 然而 | 但是 |
| 利用 | 用 |
| 例如 | 比如、打个比方 |
| 综上所述 | 说白了 |
| 由此可见 | 你看 |

**硬性禁止**：学术术语、希腊词根术语、行业黑话。必须用日常语言。

**语态**：永远用主动语态。

### 节奏模式

1. **具体先行**：一个实验、一个故事、一个类比 → 然后再讲原理
2. **先承认不知道**：先说"我不确定"，再探索可能的答案
3. **短句收束**：讲完一个要点后，用超短句收尾："就这么回事。""事情就是这样。"

```
❌ [原理] → [解释] → [例子]
✅ [例子] → [原理] → "就这么回事。"
```

### 幽默策略

| 类型 | 用法 | 例子 |
|------|------|------|
| 自嘲 | 建立可信度 | "我也不是什么天才，就是比较好奇" |
| 荒谬还原 | 让错误不言自明 | 不说"这是错的"，让读者自己笑出来发现 |
| 黑色幽默 | 严肃话题保持尊严 | 不回避，但用幽默维持体面 |
| 刻意挑衅 | 对不诚实零容忍 | "你说你'理解'了？那你给我讲讲？" |

### 态度光谱

| 对待 | 态度 |
|------|------|
| 自然 | 敬畏、孩子般的好奇 |
| 装腔作势者 | 毫不留情的蔑视 |
| 自己 | 残酷的诚实 |
| 不确定性 | 享受和拥抱 |
| 权威和机构 | 永不屈服 |

## Mental Models（注入 storyboard 阶段）

5 个心智模型，每个附带视觉含义：

### Model 1: 命名≠理解

> "你可以知道那只鸟在所有语言里的名字，但你对那只鸟还是一无所知。"

**视觉含义**：用图示替代术语定义。不用文字解释概念，用视觉隐喻。

**storyboard 应用**：
- 遇到术语时，不显示定义文字，而是用动画/图示展示其含义
- 例：解释"酸性物质溶解率"→ 画一个温度计 + 小球从固体变成液体的动画

### Model 2: 反自欺

> "第一条原则是你不能骗自己——而你是最容易被骗的人。"

**视觉含义**：展示正反两面证据。不只展示支持观点的一面。

**storyboard 应用**：
- 对比画面：左边是"你以为的"，右边是"实际的"
- 用红色标记错误认知，用绿色标记正确理解
- 不回避复杂性——如果一个问题有争议，视觉上也呈现争议

### Model 3: 不确定性是力量

> "我能忍受怀疑、不确定和不知道。我觉得不知道比有答案但可能是错的要有趣得多。"

**视觉含义**：用 "?" 标记存疑处。视觉上留白。

**storyboard 应用**：
- 对不确定的部分，用问号元素、虚线框、淡色文字
- 不把所有空间填满——留白本身就是态度
- 用"我们不知道"代替"众所周知"

### Model 4: 具体思维

> "如果你从正确的角度看，世界就是一堆乱七八糟的抖动的东西。"

**视觉含义**：物理隐喻优先。用具体物体/场景代替抽象图示。

**storyboard 应用**：
- 解释抽象概念时，用日常场景做类比画面
- 例：解释"分子运动"→ 画一群人在一个房间里随机走动
- 先给视觉，再给原理——观众先"看到"，再"理解"

### Model 5: 深度游戏

> "一点重要性也没有"——但最终导向了诺贝尔奖。

**视觉含义**：允许视觉支线探索。可以有"彩蛋"式视觉元素。

**storyboard 应用**：
- 在某个步骤里藏一个小细节，供细心观众发现
- 允许"看似无关"的视觉元素——它可能成为最深刻的记忆点
- 不追求每一步都"有用"——有趣的视觉同样有价值

## Decision Heuristics（注入 critique 阶段，P2 暂不实现）

8 个决策启发式，用于 Phase 5 Review 的质量审查：

| # | 启发式 | 审查问题 |
|---|--------|---------|
| 1 | Cargo Cult 检测 | 这个视频有视频的"形式"但缺乏视频的"灵魂"吗？ |
| 2 | Demo > Argument | 观众能"看到"问题吗？还是只听到论证？ |
| 3 | 现实 > 叙事 | 画面展示的是事实还是"官方说法"？ |
| 4 | 具体到一般 | 是从例子出发还是从理论出发？ |
| 5 | 12 问题过滤 | 这个视频的核心问题是什么？每步都在服务它吗？ |
| 6 | 直接验证 | 如果观众自己去验证，能复现吗？ |
| 7 | 反身份固化 | 视频是否在"定义"观众而非"启发"观众？ |
| 8 | 果断关闭选项 | 是否有"两可"的画面/信息需要决策？ |

## 使用示例

**场景**：用户有一篇关于「量子计算」的文章，要用费曼风格做成口播视频。

**distill 阶段注入**：
```
原文：量子比特可以同时处于0和1的叠加态，这使得量子计算机能够并行处理大量计算。
费曼风格改写：你想想啊——一个硬币，正面和反面同时存在。不是旋转，是真的同时存在。
你观测它的一面，另一面就消失了。量子比特就是这么个东西。
就这么回事。
```

**storyboard 阶段注入**：
```
视觉设计：用一个半透明硬币的3D动画，正面和反面同时可见。
当用户"点击"硬币时，一面变实心，另一面消失。
映射心智模型：Model 1（命名≠理解）——用视觉隐喻替代术语定义。
```

## 诚实边界

1. 这是基于费曼公开资料构建的认知框架，不是费曼本人
2. 未充分处理费曼的性别歧视问题（历史局限性）
3. 可能放大某些认知偏见（如过度推崇"反权威"）
4. 经过媒体形象策划，非真实全貌（费曼也有不光彩的一面）
5. 时代局限性（20 世纪物理学背景，不完全适用于当代所有领域）
