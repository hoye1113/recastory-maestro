## 信息保留度规则

**公式**：`len(script.md) / len(article.md) >= 0.6`（中文按字符数；代码块/Markdown 标记/引用源不计入）。**优先级高于所有原则**。

**禁止压缩**：关键数据/数字、具体案例/故事、多步论证中间步骤、反方观点/限制/例外。
**允许压缩**：套话开场、重复总结、学术性铺垫、引文书目、客套结语。
原文过长（5000+ 字做 3 分钟视频）→ 告知用户拉长视频或拆多集。

## script.md 格式规范

| 规则 | 要求 |
|---|---|
| 单句长度 | <= 20 字（B 站）；抖音 <= 12；YouTube <= 30；知乎 <= 25 |
| 人称 | 第二人称"你/我们"，禁用"用户/读者" |
| 开头 | 3 秒钩子（悬念/反差/利益）；禁"大家好"/"本期视频"/"在开始之前"/假共情/自我加权 |
| 节奏 | `---` 分隔每个完整想法；`---` 之间 = outline 一个 step |
| 数字 | 非核心→感受；核心冲击数字保留原值 |
| 结构词 | 禁用"首先/其次/最后"，融进过渡或砍掉 |
| 举例 | 从"我"出发；抽象→具体 |
| 禁用项 | emoji、括号补充、书名号引文（改"我看到一份报告说"） |
| 语速 | 中文 ~250 字/分钟；~4 字/秒 |

## outline.md 格式规范

**顶部 metadata**（引用块）：`主题`（theme-id）、`总时长`（口播估时）、`章节数`（N 章 / M 步）。

**章节标题**：`## N. <id> — <title>（<S> steps · ~<T>s）`

| 部分 | 规则 |
|---|---|
| `N` | 1-indexed，对齐 chapters.ts |
| `<id>` | 小写+连字符 → React key / 文件夹 `0N-<id>/` / 音频子目录 |
| `<S> steps` / `~<T>s` | step 总数 / 口播估时（字数/4） |

音频：`public/audio/<id>/<step-N>.mp3`（1-indexed）

**信息池 + 双源原则**：每章首段从 article.md 抽细节（数字/引用/案例/对比），格式 `- <类型>：<内容> —— 来源 article §X / Lxx`。script.md 定节拍（按 `---` 切），article.md 定画面密度。

**Step 列表**：`- step N (~Ts) — <屏幕内容>`。`(~Ts)` 必填（字数/4，3~10s），屏幕内容一句话 <=1 行，不写动画/时长/实现手段。每章末尾可附 1~3 句口播节选。

**章节切分**：每章 3~8 步（慢节奏 2~3，信息密集 8~10），总时长/30s ≈ 章节数，每章一个聚焦主题。

## AI 味速查表

| 类 | 模式 | 典型句式 | 修正 |
|---|---|---|---|
| 1. 假共情 | 开口就演"我懂你" | "我知道你肯定很纠结"/"你是不是也有这种感觉" | 直接抛事实/钩子 |
| 2. 假深刻 | 转折词包装平庸观点 | "恰恰相反"/"这反而是它可怕的地方"/"正是因为" | 去掉转折直接说结论 |
| 3. 自我标榜 | 说正事前先加权 | "我必须认真说"/"颠覆认知"/"你一定要听完" | 直接说 |
| 4. 万能模板 | AI 高频指纹 | "说白了"/"本质上"/"底层逻辑"/"一句话总结"/"归根结底" | 删掉或直接说内容 |
| 5. 排比堆砌 | >=3 句结构相同 | "不躲，不藏，不绕，不逃"/"高效，精准，强大" | 保留 1~2 个，其余砍掉 |

**AI 高频词**：说白了/本质上/底层逻辑/恰恰/反而/正是因为/归根结底/在某种程度上/换句话说/以上就是本期内容/希望对你有帮助/感谢观看

## 反模式速查

### DS（Distill-Style 口播风格）

| 编号 | 反模式 | 判定 |
|---|---|---|
| DS-001 | 信息保留度 < 60% | 字符比不达标，关键事实/案例/论证链消失 |
| DS-002 | 长句未拆 | 单句 > 20 字（B 站基准） |
| DS-003 | 第三人称疏离 | "用户""读者"而非"你""我们" |
| DS-004 | 无钩子开头 | 前 3 秒无悬念/反差/利益 |
| DS-005 | 结构词堆砌 | "首先/其次/最后"或"下面进入第二部分" |
| DS-006 | 数字未翻译 | 非核心数字直接念百分比/大数字 |

### CD（内容提炼结构）

| 编号 | 反模式 | 判定 |
|---|---|---|
| CD-001 | 大纲层级超过 4 层 | 扁平化或拆分为多个视频 |
| CD-002 | 单章节字数 >500 字（短视频模式） | 拆分为 2-3 个子章节 |
| CD-003 | 口播脚本包含书面语 | "综上所述""由此可见"等需替换为口语化表达 |
| CD-004 | 信息密度过低 | 关键词覆盖率 <30% |
| CD-005 | 缺乏 Hook | 前 10 秒无吸引力语句 |
| CD-006 | 视角未注入 | 指定了 perspective 但 script 风格中性 |

### SL（风格层）

| 编号 | 反模式 | 判定 |
|---|---|---|
| SL-001 | 假共情 | "我知道你""你是不是也""我能理解" |
| SL-002 | 假深刻 | "恰恰/反而/正是"包装，去掉后意思不变 |
| SL-003 | 自我标榜 | "我必须认真说""颠覆认知""你一定要听完" |
| SL-004 | 万能模板 | "说白了/本质上/底层逻辑/一句话总结/归根结底" |
| SL-005 | 排比堆砌 | 连续 >=3 句结构相同 |
| SL-006 | 套话结尾 | "以上就是本期内容/希望对你有帮助/感谢观看" |
